Flagship project

Factory Cost Control Tower for a Bottling Line

End-to-end manufacturing analytics case study that explains which downtime causes, shifts, and SKUs drive the highest cost per good unit.

  • Python
  • DuckDB
  • Pandas
  • Power BI
  • Manufacturing KPIs
Observation window Aug 29 to Sep 3, 2024
Scope 38 batches, 6 SKUs, 3 shifts
Output Curated marts, QA tests, Power BI build pack
View Repo View Dashboard Screens

Project KPIs

R$ 0.0669 Average cost per good unit
23.1 h Total downtime across the sample
381 min Setup and calibration loss
+20% Swing shift cost gap vs day shift

Problem

Problem

Plant leadership could see output volume, but not the economic effect of downtime by shift, SKU, or work pattern. Improvement efforts risked becoming generic labor pressure instead of targeted action.

Approach

Approach

I used the Maven bottling-line workbook, cleaned it in Python, modeled plant economics with documented reference tables, transformed the data in DuckDB, and produced Power BI-ready marts plus dashboard screens.

Model

Data Model

The model uses fact_batch_runs at batch grain, mart_root_cause at batch-by-factor grain, and mart_action_queue for ranked opportunities, supported by product, shift, operator, and downtime dimensions.

Outcome

Outcome

The modeled line averages R$ 0.0669 per good unit, setup and calibration drive the largest downtime bucket at 381 minutes, and swing shift runs at R$ 0.0726 per unit versus R$ 0.0605 on day shift.

Recommended next actions

  • Standardize setup sheets and first-article checks on swing shift.
  • Prioritize preventive maintenance around machine adjustment and failure events.
  • Pre-stage packaging and materials one batch ahead to cut material-flow loss.

Dashboard highlights

The output includes an executive overview, shift and operator benchmark, downtime root-cause analysis, and a what-if screen for wage, downtime, and loss scenarios.
