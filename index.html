Case Study: Retail Performance & Customer Insights

Problem

Retail teams lacked a consistent view of revenue, margin, and returns by region and category, and weekly reporting was manual.

Approach

Used a public retail dataset, cleaned and shaped it in Power Query, modeled a star schema, and standardized KPIs in DAX.

Data Model

FactSales + FactReturns linked to Date, Product, Store, and Customer dimensions for clean drill-downs.

Data & Metrics

Dataset includes orders, products, customers, stores, and dates. Key metrics include revenue, margin %, return rate, average order value, and repeat purchase rate.

Dashboard Highlights

Executive KPIs (revenue, margin %, return rate), trends, region/category drill-downs, and customer segments.

Automation

Planned Power BI Service refresh with optional Power Automate alerts for margin drops and return spikes.

Outcome

Single source of truth with clear KPI definitions and faster weekly reporting for stakeholders.

Improvements

Next steps: publish to Power BI Service, add RLS, incremental refresh, and a data dictionary.
